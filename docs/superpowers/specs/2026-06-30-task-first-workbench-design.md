# 任务优先工作台 UI 改造设计

## 目标

把当前“创建表单常驻左侧、任务列表与详情纵向堆叠”的工作台，改成以日常任务管理为默认入口的轻量工作台。用户打开页面后应先看到任务状态、任务列表和当前任务产物；新建任务和设置分别进入独立页面。

本轮只调整前端信息架构、布局和视觉层级，不改变后端 API、任务状态模型、创建参数或产物处理逻辑。

## 现状与问题

- 约 360–430px 的左栏长期承载完整创建表单，包含模板、运行配置、路径、高级选项和上传区，挤压任务管理空间。
- 右侧依次放置标题、四张指标卡、任务列表、任务详情，纵向滚动距离长；列表与详情缺少主从关系。
- “新建”和“设置”只控制左栏，右侧仍固定显示任务看板，页面状态不明确。
- 顶部存在多个弱价值装饰信息，主操作“新建任务”反而不在任务看板的主要操作区。
- 空状态仍提示“从左侧新建”，说明信息架构依赖旧布局。

## 开源产品调研

### Dify

Dify 把应用与工作流作为工作区核心对象，创建动作是明确入口，配置则留在应用或工作区上下文中。可借鉴点：稳定导航、对象优先、创建动作突出，不让完整配置长期占据主工作区。

参考：<https://github.com/langgenius/dify>、<https://docs.dify.ai/en/community/contribution>

### n8n

n8n 的执行记录以独立的 Executions 视图呈现，支持按工作流和状态筛选，再进入单次执行上下文处理失败或重试。可借鉴点：任务列表应是默认工作面，状态筛选应直接作用于列表。

参考：<https://docs.n8n.io/workflows/executions/all-executions/>

### Open WebUI

Open WebUI 把 Workspace 能力放在稳定导航下，Prompts、Models 等对象各有独立创建和管理页面，避免把所有表单同时铺开。可借鉴点：导航负责切换任务域，页面只承载当前任务。

参考：<https://docs.openwebui.com/features/workspace/prompts/>、<https://docs.openwebui.com/features/workspace/models/>

### Plane

Plane 以工作项为一等对象，并用筛选视图缩小当前关注范围。可借鉴点：列表与详情形成主从关系，筛选和操作贴近工作项，而不是额外堆叠大卡片。

参考：<https://github.com/makeplane/plane>

## 方案比较

### 方案 A：任务中心 + 独立新建页（采用）

- 固定轻量侧栏：任务、新建、设置三个入口。
- 默认进入任务中心；顶部只保留页面标题、状态和主操作。
- 状态统计改成紧凑筛选条。
- 桌面端使用任务列表 / 任务详情双栏布局。
- 新建任务与设置分别占用完整主内容区。

优点：直接解决空间竞争，符合用户选择的“任务管理优先”，且可以复用现有组件与数据流。缺点：创建任务需要一次页面切换。

### 方案 B：任务中心 + 新建抽屉

保留任务中心，点击新建后从右侧拉出完整表单。创建上下文切换更快，但当前表单较长，抽屉会形成第二个滚动容器，并在窄屏产生可用性问题。

### 方案 C：单页可折叠仪表盘

继续保留所有模块，通过折叠和卡片密度减轻拥挤。改动最小，但没有解决创建与管理争夺首屏的问题，仍会形成长页面。

## 采用设计

### 信息架构

工作台包含三个一级页面：

1. **任务中心**（默认）：状态筛选、任务列表、任务详情和产物。
2. **新建任务**：模板、需求、资料、输出路径、运行配置和高级选项。
3. **设置**：API 模型配置、运行能力和已保存配置。

创建成功后自动返回任务中心并选中新任务。首次无 API 配置时，从新建页跳转设置页；保存后仍由用户主动返回新建，避免新增隐式流程。

### 桌面布局

- 左侧导航宽度约 220px，保留品牌、三项导航和底部本机运行状态。
- 主内容区最大宽度不做人为限制，使用 24px 外边距。
- 任务中心顶部是单行页面标题、主按钮、刷新与清理操作。
- 四个状态改成高度约 44px 的紧凑筛选按钮，不再作为大号 KPI 卡片。
- 下方为 `minmax(320px, 0.8fr) / minmax(520px, 1.4fr)` 双栏：左侧任务列表，右侧当前任务详情。
- 列表内部滚动，详情保持独立阅读区；无任务时提供明确的新建入口。

### 响应式布局

- 1180px 以下将任务列表与详情上下排列。
- 760px 以下把左侧导航变成顶部紧凑导航，标题操作区纵向排列。
- 保留现有表单、目录选择器和产物操作的移动端规则。

### 视觉层级

- 继续使用当前绿色品牌色、语义状态色和深色模式，避免无关换肤。
- 去掉顶栏和指标区的重复卡片感，通过边框、背景和留白建立层级。
- 每页只保留一个高强调主按钮：任务中心“新建任务”、新建页“开始生成”、设置页“保存配置”。
- 保留状态胶囊，但任务行缩短状态列宽，突出任务类型、需求摘要和更新时间。

### 数据流与行为

- `App` 的视图状态扩展为 `jobs | create | settings`，默认 `jobs`。
- 现有 `refresh`、SSE、筛选、删除、取消、产物再加工逻辑不变。
- `CreateJob.onCreated` 在刷新后切回 `jobs` 并选中新任务。
- 连接失败横幅仍在主内容区顶部显示，所有页面可见。
- 筛选状态仍复用现有 `statusFilter`，不新增搜索、分页或持久化等未要求功能。

## 实现边界

### 修改

- `frontend/src/main.tsx`：调整一级导航、页面条件渲染、任务中心布局和创建成功后的视图切换。
- `frontend/src/styles.css`：重建应用壳层、导航、顶部操作区、紧凑筛选条和任务主从布局。
- `frontend/tests/`：补充静态或行为断言，覆盖默认任务页、三级导航和创建后返回任务中心。
- `docs/` 与 `progress.md`：同步信息架构与验证记录。

### 不修改

- 后端 API、数据库结构、任务状态协议、SSE 协议。
- 创建表单字段、模板内容、任务执行规则和设置数据结构。
- 发布流程、启动脚本和落地页素材。

## 验收标准

1. 打开工作台默认显示任务中心，完整创建表单不再常驻首屏。
2. 左侧导航可以明确切换任务、新建、设置三个页面。
3. 桌面宽屏下任务列表与任务详情并排呈现；窄屏下正确堆叠。
4. 点击状态筛选只影响任务列表，清除筛选可恢复全部任务。
5. 新建任务成功后返回任务中心并选中新任务。
6. 断连提示、刷新、清理、取消、删除、下载与再加工入口仍可用。
7. 前端测试、TypeScript 检查和 Vite 构建通过。
8. 使用真实浏览器检查任务中心、新建页、设置页以及 1440px、768px 两档布局，无横向溢出和明显遮挡。
